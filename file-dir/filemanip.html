Touch

The touch utility "makes contact" with a given file location. If the specified file exists, using the touch utility updates the file's timestamp to the current time. If the file does not exist, the touch utility creates a file at that location.

Syntax

touch [path]

Examples

touch file1.txt
touch new_list
touch data/2008/november/week1

Exercise

Use the change directory and list utilities to find a text file. (If you are using a distribution of Unix that includes a full GUI, you can instead create a new file for this purpose using a text editor.) Use the long listing (-l) option of the list utility to see the timestamp on the file. Touch the file and then use the list utility with the long listing option again to see the new timestamp. The file you touched should be updated with the current time.

solution
ls -l
touch [file]
ls -l

Copy

The copy utility creates a duplicate of the file in the first path at the second path.

Syntax

cp [options] [path 1] [path 2]

Examples

cp files/file1 files/file1copy
cp data_template.txt new_data.txt

Exercise

Create a copy of the file you used in the previous exercise and name the copy example1.txt. Use the list utility to check your work.

solution
cp [original file] example1.txt
ls

If you wish, the following command can be used to check that the contents of the new file are the same as the original:

cat example1.txt

The output should be the same as the contents of the file you copied. This utility will be covered in more detail in the section of the tutorial about viewing files.

Move

The mv utility is used to move or rename files. It creates a copy of the file at the first location in the second location, then deletes the original.

Syntax

mv [path 1] [path 2]

Examples

mv files/file1 files/file2
mv typo.txt type.txt
mv new_data data/new_data

Exercise

Use the move utility to change the name of the copy you created in the pervious exercise (example1.txt) to example2.txt. Then list the files in that directory. You should be able to see example2.txt but not example1.txt.

solution
mv example1.txt example2.txt
ls

Next, move example2.txt one folder higher in the directory structure. (That is, move it into its parent folder.) List the contents of the directory that example2.txt used to be in and the directory you moved it to. Only the latter should contain a file named example2.txt.

solution
mv example2.txt ../example2.txt
ls
ls ..

Remove

The rm utility deletes the file at the specified location.

Syntax

rm [path]

Examples

rm files/file.txt
rm junk_data

Exercise

Use the remove utility to delete example2.txt. Then, use the list utility to make sure you deleted it successfully.

solution
rm ../example2.txt
ls ..

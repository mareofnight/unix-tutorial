Viewing Permissions

The long listing option of the list utility includes permissions information in its output.

Syntax

ls -l[other options] [directory]

Examples

ls -l
ls -l courses/409

Exercise

Use the touch utility to create a new file named example in your working directory. Use the list utility to print this file's permissions information.

solution
touch example
ls -l

Changing Permissions

The chmod utility changes a file's permissions. It can be used with either numeric or character permissions codes.

Syntax

To set permissions with numbers:
chmod [permissions number] [path]
To set permissions with characters:
chmod [ [user group character(s)]=[permissions character(s)] separated by commas] [path]
To add permissions with characters:
chmod [ [user group character(s)]+[permissions character(s)] separated by commas] [path]
To revoke permissions with characters:
chmod [ [user group character(s)]-[permissions character(s)] separated by commas] [path]

Examples

chmod 755 /www
chmod u=rwx,og=rx /www
chmod 644 index.html
chmod u=rw,og=r index.html
chmod u-w static_document.txt
chmod g+rwx,o+rx shared_folder

Exercise

Set the file so that it is read only for you and no other users can see it. Print the permissions information using the list utility. Then, change its permissions again to grant write permissions to yourself and read permissions to other users in your group. Print its permission information again to see the changes.

solution
chmod 400 example

or

chmod u=r,go= example
ls -l
chmod u=rw,g=r example
or
chmod 640 example
ls -l

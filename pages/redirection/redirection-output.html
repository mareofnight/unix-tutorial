Saving Files with Redirection

The simplest method of redirecting a utility's output functions as a "save as", either creating a new file or replacing the contents of an old one with the utility's output.

Syntax

[utility] [options] > [output file]

Examples

Exercise

Use redirection and the concatonate utility to save a new file named output containing several lines of text. To do this, use the default input (the keyboard) and redirect the output. When you are done typing the input text, use the keystroke ^D to run the command. When you are done, use one of the file viewing utilities to check your work.

solution
cat > output

Appending Files with Redirection

It is also possible to use redirection to add more content to the end of a file without deleting the old content. To do this, use a double right angle bracket instead of a single one.

Syntax

[utility] [options] >> [output file]

Examples

who >> all-logins.txt

Exercise

Use redirection and the concatonate utility to add a few more lines to the file you created. Use a file viewing utility to check your work. You should see the text you entered in the previous exercise, followed by the new text.

solution
cat >> output

Redirecting Both Input and Output

It is possible to use both input and output redirection in the same command. This can be used to process data and save the results in a single command while preserving the original file.

Syntax

[utility] [options] < [input file] > [output file]

Examples

sort < prioritized-reading-list > alphabetized-reading-list

Exercise

Use redirection and the concatonate utility to make a copy of the input file you created for the input redirection exercise. Check your work; the two files should have identical contents.

solution
cat < input-file.txt > output-file.txt

Pico is a text editor included in most Unix distributions. Its main use is making very simple changes to text files; most people prefer to use vi or emacs for extensive editing.

The following command will launch pico:

pico

If you are using a Unix-like operating system derived from the GNU project, pico will launch nano instead of pico. Nano is extremely similar to pico, but with some additional features. It was developed as a replacement for pico because pico's licinse was not a free software licinse.

A note on syntax

In Unix's help features and most resources on Unix, including this tutorial, the ^ character represents holding down the control key while pressing the key immediately following it. Letters combined with ^ should be typed without holding down shift, as though typing them in lower case, even if uppercase characters are used to represent them.

Cut

^K cuts (copies and deletes) the line of text the cursor is in.

Syntax

^K

Exercise

Use the touch utility to create a new file and open the file in Pico. Type a few lines of text. (Include carriage returns; line breaks enforced by the width of the terminal are not considered line breaks for the purposes of copying text.) Copy one of the lines.

Un-

The function of ^U depends on the last keystroke used. In general, it does the opposite of the previous keystroke. For example, after cutting text, ^U is used to paste. The current function of ^U will be displayed somewhere in pico's interface.

Syntax

^U

Exercise

"Uncopy" (that is, paste) the text that you copied in the previous exercise.

Turning the Pages

In some Unix distributions, the pico interface does not scroll, and text is instead displayed in pages. ^V and ^Y are used to navigate through long files that require multiple pages.

Syntax

To go to the next page:
^V
To go to the previous page:
^Y

Exercise

Add some nonsense characters to the file, including carraige returns, to make the text extend past the first page. Observe how your distribution of Unix handles this; some have scroll bars in their text editors, and some do not. Use the previous and next commands to flip back and forth through the pages.

Save

Pico's save feature functions similarly to those of most programs. After typing the ^O keystroke, you will be given a prompt with the name the file will be saved as. By default, this is the file's original name, but you can also change it to save a copy.

Syntax

^O

Exercise

Save your file under a different name than the original.

Exit

^X is used to exit pico and return to the terminal. If changes to the file have not been saved,a prompt will give you a last chance to save before exiting.

Syntax

^X

Exercise

Exit pico, and then open your original file in pico. It should be empty again because you didn not save your changes to this file name. Exit pico again and then use it to open the copy you saved. This file should containt the text you entered.

Redirection: Piping

Piping redirection in which the output of one utility becomes the input for another. Using multiple programs to process a file using other redirection methods would require saving a file for each intermediate step and entering a different command for each program involved. Piping makes it possible to execute multiple utilities in a single command.

How to Use Piping

The | character is used to indicate a pipe between utilities. Piping can also be combined with input and output redirection. It is possible to use redirection to specify an input file for the first utility, and/or an output file for the last utility.

Syntax

[first utility] [options] [ < input file] | [second utility] [options] | [more utilities...] | [last utility] [options] [ > output file]

Examples

cat list.txt | sort
cat list.txt | sort
who | sort | cat > sorted_users.txt

Exercise

This exercise will use the word count (wc) and concatonate utilities. The word count utility outputs the number of lines, words and characters in its input.

Create a new file with some text in it or make a copy of an existing file and name it pipe-example.txt. Use piping to print the line, word and character counts for this file to the terminal. Then, write a different command that uses piping and redirection to save these values in the file pipe-example-counts.txt. Use the cat utility to check your work when you are done.

solution
cat pipe-example.txt | wc
cat pipe-example.txt | wc > pipe-example-counts.txt
cat pipe-example-counts.txt

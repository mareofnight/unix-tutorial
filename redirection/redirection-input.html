Redirection in Unix is taking input of a utility from or sending output to a location other than the default. This can be used to save to or read from files using utilities that ordinarily do not save or read files, or to use the ouput of one utility as input for another.

The concatonate utility will be used as an example of how redirection works in Unix because its simplicity makes it easy to observe. Concatonate allows input files to be specified as arguments, as described in the viewing files section of this tutorial. For the purposes of demonstrating redirection, this feature should not be used. When no files are specified as arguments, concatonate will use the keyboard as its input location and print its output to the terminal.

How to Redirect Input

Redirecting input is useful for processing large files without having to re-type them.

Syntax

[utility] [options] < [input file]

Examples

sort < unalphabetized_list.txt

Exercise

Create a text file named input-file.txt with a few lines of text in it using one of the methods covered earlier in this tutorial. Use the concatonate utility to print this file. The output should be the same that you would see if specifying the input file as an argument.

solution
cat < input-file.txt

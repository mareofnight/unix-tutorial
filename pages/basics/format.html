Formatting Commands

previous

Commands in Unix use the following format:

[utility] [options] [path1] [path2]

A Unix utility is a program included in the Unix OS. Each command entered into the Unix terminal starts with the name of the utility being used. In some cases, such as when starting a program without opening a particular file, this is the only necessary part of the command.

Options are added next. They are used to alter the way a utility runs. There are two different formats for options: a double dash followed by several characters, or a single dash followed by a single character. Some options can be written both ways. For example, in the ls utility (used to list the contents of a directory), the options --all and -a both show hidden as well as non-hidden files.

If the command will be interacting with files or directories, their paths come next. Usually, the path(s) will be relative to the working directory, the first path will be the input, and the second will be the output. If no paths are specified, defaults will be used instead. The default output is the terminal window, and the default input is the keyboard.

Spaces are used to separate each part of a command. Because spaces are used as delimiters, they should not be used in names of files. (Underscores and camel casing can be used instead.) After typing a command, press enter to finish and run it.

Some features of unix, such as redirection, require additional information in the command. Formatting for use with these features is covered in the relevant sections of the tutorial.
